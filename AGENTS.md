# הנחיות פיתוח ולוגיקת המערכת - סמינר שצ'רנסקי בית שמש

מסמך זה מגדיר את דרישות המערכת, חוקי התקציב, והנוסחאות השעתיות של **מערכת דיווח ובקרת שכר ותקציב - סמינר שצ'רנסקי בית שמש (שנת לימודים תשפ"ז)**.
אין לבצע שינויים בקוד הסותרים את הלוגיקה, הנוסחאות, או הרשאות התפקידים המפורטות להלן.

---

## 1. לוגיקת חישוב התקציב ונוסחאות שכר מותאמות

על המערכת לתמוך בארבע צורות תשלום מוגדרות מראש, כאשר כל אחת מחשבת את "סה"כ שנתי שכר מורה" בצורה שונה בהתבסס על שעות שבועיות (ש"ש), מפגשים/חודשים, ותעריף שעה בסיסי:

1. **תקן (+45% תקורות מעביד)**:
   - **בסיס החישוב**: `ש"ש * מפגשים` (מפגשים/חודשים כפי שהוזנו — ללא כפל קבוע ב-30 שבועות).
   - **עלות מעביד (תעריף לשעה כולל תקורות)**: `תעריף לשעה * 1.45`.
   - **סה"כ שנתי שכר**: `ש"ש * מפגשים * תעריף לשעה * 1.45` (בין אם התעריף חודשי או שבועי).

2. **שכר מרצים (+30% תקורות מעביד)**:
   - **שעות שנתיות לתלמידות**: `ש"ש * מפגשים` (לפי מספר מפגשי הוראה שנתיים בפועל).
   - **עלות מעביד (תעריף לשעה כולל תקורות)**: `תעריף לשעה * 1.30`.
   - **סה"כ שנתי שכר**: `שעות שנתיות לתלמידות * עלות מעביד לשעה`.

3. **קבלה (+18% תקורות מעביד)**:
   - **שעות שנתיות לתלמידות**: `ש"ש * מפגשים`.
   - **עלות מעביד (תעריף לשעה כולל תקורות)**: `תעריף לשעה * 1.18`.
   - **סה"כ שנתי שכר**: `שעות שנתיות לתלמידות * עלות מעביד לשעה`.

4. **קבלת פטור (0% תקורות מעביד)**:
   - **שעות שנתיות לתלמידות**: `ש"ש * מפגשים`.
   - **עלות מעביד (תעריף לשעה כולל תקורות)**: `תעריף לשעה * 1.00` (ללא תוספת).
   - **סה"כ שנתי שכר**: `שעות שנתיות לתלמידות * עלות מעביד לשעה`.

---

## 2. חוקי שלמות נתונים (Data Integrity) ואימותים

1. **אימייל וטלפון חובה**: לא ניתן לשמור שורת מורה במערכת ללא הזנת מספר טלפון תקין וכתובת אימייל במבנה חוקי (`name@domain.com`).
2. **אימות תפקידים**:
   - **מנהלת סמינר (Director)**: בעלת הרשאה מלאה לאשר/לבטל אישורי שכר של מורים (`is_approved`), לנהל סיסמאות רכזות, ולהגדיר את חיבור הענן.
   - **מזכירה (Secretary)**: בעלת הרשאה להוסיף, לערוך ולמחוק שורות, להכין חוזים מתוך דיווחים מאושרים, ולצפות בדוח התקציב הסופי. אינה מורשית לאשר שורות שכר.
   - **רכזת מסלול (Coordinator)**: מוגבלת אך ורק למסלול ההתמחות שלה. מורשית להוסיף ולערוך משרות אך ורק השייכות למסלול הפעיל שלה.

---

## 3. אינטגרציה לענן ומסד נתונים (SUPABASE / POSTGRESQL)

על מנת לעבוד בצורה מבוזרת ומסונכרנת, המערכת תומכת בחיבור ישיר ל-**Supabase** (או כל מסד נתונים PostgreSQL אחר) דרך שרת Express אחורי. 

### תסריט הרצה להקמת מסד הנתונים ב-Supabase (SQL Editor):
יש להריץ את ה-SQL הבא בלוח הבקרה של Supabase על מנת ליצור את הטבלה הנדרשת:

```sql
-- יצירת טבלת דיווחי שכר ותקציב
CREATE TABLE IF NOT EXISTS salary_records (
    id SERIAL PRIMARY KEY,
    track TEXT NOT NULL,                  -- התמחות / מסלול
    year TEXT NOT NULL,                   -- שנה (יג, יד, יג+יד)
    teacher_name TEXT NOT NULL,           -- שם המורה
    subject TEXT NOT NULL,                -- שם המקצוע
    semester TEXT NOT NULL,               -- סמסטר / מחזור
    payment_method TEXT NOT NULL,         -- צורת תשלום (תקן, שכר מרצים, קבלה, קבלת פטור)
    shash DECIMAL(10,2) NOT NULL DEFAULT 0, -- ש"ש
    meetings INTEGER NOT NULL DEFAULT 0,  -- חודשים / מפגשים
    total_hours INTEGER NOT NULL DEFAULT 0, -- שעות שנתיות לתלמידות (מחושב)
    rate DECIMAL(10,2) NOT NULL DEFAULT 0, -- תעריף לשעה
    employer_overhead DECIMAL(10,2) NOT NULL DEFAULT 0, -- עלות מעביד כולל תקורות
    total_annual DECIMAL(10,2) NOT NULL DEFAULT 0, -- סה"כ שנתי שכר מורה
    tz TEXT,                              -- ת.ז מורה
    phone TEXT NOT NULL,                  -- טלפון מורה (חובה)
    email TEXT NOT NULL,                  -- אימייל מורה (חובה)
    is_approved BOOLEAN NOT NULL DEFAULT FALSE, -- סטטוס אישור שכר ע"י המנהלת
    is_contract_ready BOOLEAN NOT NULL DEFAULT FALSE, -- סטטוס הכנת חוזה
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- יצירת אינדקסים לחיפוש מהיר ופילוח
CREATE INDEX IF NOT EXISTS idx_salary_records_track ON salary_records(track);
CREATE INDEX IF NOT EXISTS idx_salary_records_teacher ON salary_records(teacher_name);

-- הגדרת הרשאות קריאה וכתיבה (במידת הצורך עבור פיתוח חופשי)
ALTER TABLE salary_records ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow public read/write to salary_records" ON salary_records FOR ALL USING (true);
```

### חיבור המערכת:
המערכת משתמשת במשתנה סביבה בשם `DATABASE_URL` המכיל את מחרוזת החיבור (Connection String) של Supabase, למשל:
`postgresql://postgres:YOUR_PASSWORD@db.YOUR_PROJECT_REF.supabase.co:5432/postgres`
ממשק המשתמש יאפשר גם הזנת כתובת API חלופית או עבודה במצב מקומי מסונכרן.
